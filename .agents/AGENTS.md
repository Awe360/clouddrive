# CloudDrive — AI Agent Context

## Project Overview
CloudDrive is a full-stack SaaS file management platform (mini Dropbox/Google Drive) built to teach AWS fundamentals. It uses a Node.js + React stack deployed on AWS using Terraform and Ansible.

## Repository Layout
```
full-app/
├── backend/              # Node.js Express REST API (port 3000)
├── frontend/             # React 18 + TypeScript + Vite (port 5173)
└── infrastructure/
    ├── terraform/        # AWS IaC — all resources as code
    └── ansible/          # EC2 configuration automation
```

## Tech Stack
- **Frontend**: React 18 + TypeScript + Vite + React Router + Axios
- **Backend**: Node.js 20 + Express + mysql2 + @aws-sdk/client-s3 + JWT + bcryptjs + multer
- **Database**: MySQL 8.0 on AWS RDS (db.t3.micro, free tier)
- **File Storage**: AWS S3 (standard, with versioning + lifecycle rules)
- **Compute**: AWS EC2 (t2.micro Ubuntu 22.04, free tier)
- **Reverse Proxy**: Nginx (proxies /api → Node:3000, serves React static files)
- **Process Manager**: PM2
- **IaC**: Terraform >= 1.6, AWS provider ~> 5.0
- **Config Mgmt**: Ansible >= 2.12

## AWS Services Used
- **IAM**: EC2 instance role with S3 policy (no hardcoded keys)
- **VPC**: 10.0.0.0/16, 2 public subnets (10.0.1-2.0/24), 2 private subnets (10.0.3-4.0/24)
- **Security Groups**: sg-alb (80/443), sg-ec2 (3000 from alb, 22), sg-rds (3306 from ec2)
- **EC2**: t3.micro, public subnet, Ubuntu 22.04 (t3.micro is free tier on newer AWS accounts)
- **S3**: clouddrive-files-{env}-{random}, SSE-AES256, versioning, lifecycle (GLACIER after 365d, temp/ delete after 1d)
- **RDS**: MySQL 8.0, db.t3.micro, private subnets, no public access, 7-day backup
- **ALB**: application LB (optional, disabled by default via `create_alb = false`)

## Key Design Decisions
1. **No hardcoded AWS keys**: EC2 uses IAM Instance Profile → S3 access via metadata service
2. **Upload strategy**: Files upload THROUGH Node.js API (not presigned PUT), stored in S3 at `users/{user_id}/{uuid}-{filename}`
3. **Download strategy**: Presigned GET URLs (15-min expiry) generated by API — S3 bucket is private
4. **No NAT Gateway**: Skipped to stay free tier (EC2 in public subnet, RDS in private subnet within same VPC)
5. **Terraform state**: Local (no S3 backend) — simple for solo learning
6. **Auth**: Stateless JWT (7-day expiry), bcrypt rounds=12

## Database Schema
```sql
users:  id, name, email, password, created_at
files:  id, user_id (FK), filename, s3_key, size_bytes, mime_type, uploaded_at
```

## API Structure
```
POST   /auth/register
POST   /auth/login
GET    /auth/me              (JWT required)

POST   /files/upload         (JWT + multipart/form-data)
GET    /files                (JWT)
GET    /files/stats          (JWT)
GET    /files/:id/download   (JWT) → returns presigned S3 URL
DELETE /files/:id            (JWT)

GET    /users/profile        (JWT)
PATCH  /users/profile        (JWT)

GET    /api/health           (public — used by ALB health check)
```

## Environment Variables (backend/.env)

```
DB_HOST, DB_PORT, DB_USER, DB_PASSWORD, DB_NAME
AWS_REGION, S3_BUCKET_NAME

```

## Deployment Flow
1. `terraform apply` → creates VPC, EC2, RDS, S3, IAM, Security Groups
2. Update `infrastructure/ansible/inventory.ini` with EC2 public IP
3. `ansible-playbook site.yml` → installs Node.js, Nginx, PM2, deploys app
4. SSH into EC2, run `schema.sql` against RDS, configure `.env`
5. `pm2 restart clouddrive-api`

## Free Tier Notes
- t2.micro EC2: 750 hrs/month (12 months)
- db.t3.micro RDS: 750 hrs/month (12 months)
- S3: 5GB free
- ALB is disabled by default (costs ~$16/mo): set `create_alb = true` to enable
- Run `terraform destroy` when not using to avoid charges

## Terraform Module Structure
```
modules/
├── vpc/    → VPC, subnets, IGW, route tables, security groups
├── iam/    → EC2 role, S3 policy, instance profile
├── s3/     → Bucket, versioning, encryption, lifecycle, CORS
├── rds/    → DB subnet group, MySQL instance
├── ec2/    → Ubuntu instance, user data
└── alb/    → ALB, target group, listener (count=0 when disabled)
```
